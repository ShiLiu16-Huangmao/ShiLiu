## 1. 选择你的工具

有两款手机 App 可以实现"拍照后自动上传到猫咪相册"，任选一个即可：

| App | 适用平台 | 特点 |
|-----|----------|------|
| **PicHoro**（图床神器） | Android | 功能全面，支持批量上传，界面直观 |
| **PicPlus** | iPhone / Android | 跨平台，界面简洁，适合初学者 |

> 推荐 Android 用 PicHoro，iPhone 用 PicPlus。两款操作逻辑几乎一样。

### 下载方式

- **PicHoro**：酷安、Google Play、或作者 GitHub 发布页
- **PicPlus**：App Store 搜 "PicPlus" / Google Play 搜 "PicPlus"

## 2. 获取 GitHub Token

无论用哪款 App，都需要一个 GitHub Token（相当于一把"钥匙"）。

### 步骤图解

**第 1 步**：手机浏览器打开 https://github.com/settings/tokens

> 如果提示登录，先登录你的 GitHub 账号。没有账号的话先去 https://github.com 注册一个（免费）。

**第 2 步**：找到 **Personal access tokens (Tokens classic)** 这一项，点进去。

**第 3 步**：点击页面右上角的 **Generate new token**，选择 **Generate new token (classic)**。

**第 4 步**：填写 Token 信息

```
Note（备注）：       猫咪相册上传              ← 随便写，方便日后识别
Expiration（过期）：  No expiration             ← 选"永不过期"
```

**第 5 步**：勾选权限。在权限列表中，找到 **repo** 这一项，把它勾上。

> "repo" 权限代表"读/写仓库内容"，这正是上传图片所需的。

**第 6 步**：拉到页面最底部，点击绿色的 **Generate token** 按钮。

**第 7 步**：🔑 **立刻复制**屏幕上显示的 Token！

> Token 以 `ghp_` 开头，像这样：`ghp_aBcDeFgHiJkLmNoPqRsTuVwXyZ12345`
>
> ⚠️ **这个 Token 只会显示这一次**。如果没复制到，只能删除旧的重新生成。
>
> 建议先粘贴到手机备忘录里保存。

## 3. PicHoro 配置教程

### 安装

酷安搜索 **PicHoro**（或"图床神器"），下载安装。

### 配置 GitHub 仓库

**第 1 步**：打开 PicHoro，进入「**设置**」或「**图床管理**」页面。

**第 2 步**：点击「**添加图床**」或右下角的 **＋** 按钮。

**第 3 步**：在图床类型中选择 **GitHub**。

**第 4 步**：填写配置信息：

```
┌──────────────────────────────────────┐
│  图床名称     猫咪相册                 │
│  Token        ghp_你的Token            │  ← 粘贴第 2 步复制的 Token
│  用户名       你的GitHub用户名          │  ← 比如 xiaoming
│  仓库名       你的仓库名                │  ← 比如 cat-gallery
│  分支         main                     │
│  存储路径     images/                  │  ← 关键！图片上传到这个目录
│  自定义域名   不填                      │
└──────────────────────────────────────┘
```

**第 5 步**：点击「**保存**」或「**测试**」验证配置是否正确。

### 设置为默认图床

回到图床列表，将刚才添加的 **猫咪相册** 设为默认图床（长按或点击设置）。

## 4. PicPlus 配置教程

### 安装

App Store 搜索 **PicPlus**，下载安装。

### 配置 GitHub 仓库

**第 1 步**：打开 PicPlus，点击底部「**设置**」标签。

**第 2 步**：找到「**图床设置**」，点击进入。

**第 3 步**：点击「**添加图床**」。

**第 4 步**：选择 **GitHub**。

**第 5 步**：填写配置：

```
仓库所有者：   你的GitHub用户名
仓库名称：     你的仓库名
分支：         main
Token：        ghp_你的Token
保存路径：     images/
```

**第 6 步**：保存并设为默认。

## 5. 上传照片

配置完成后，每次上传只需三步：

### 使用 PicHoro 上传

**方式一：从相册选择**

1. 打开 PicHoro，进入「**上传**」页面
2. 点击「**选择图片**」，从手机相册勾选猫咪照片（可多选）
3. 点击「**上传**」按钮
4. 等待上传完成，出现 ✅ 即成功

**方式二：拍照上传**

1. 打开 PicHoro，点击「**拍照**」
2. 对准猫咪，咔嚓 📸
3. 点击「**使用照片**」
4. 自动开始上传

### 使用 PicPlus 上传

**方式一：从相册选择**

1. 打开 PicPlus，点击首页的「**＋**」或「**上传**」
2. 选择「**从相册选取**」
3. 勾选猫咪照片，点击「**完成**」
4. 等待上传完成

**方式二：拍照上传**

1. 打开 PicPlus，点击「**拍照上传**」
2. 拍好照片后点击「**使用照片**」

## 6. 上传后发生了什么

这是最省心的部分 — **你不需要做任何额外操作**，一切自动完成：

```
你点击"上传"
    │
    ▼
📱 App 把图片通过 GitHub API 传到仓库的 images/ 目录
    │
    ▼
🔔 GitHub 检测到 images/ 目录有新文件
    │
    ▼
🤖 GitHub Actions 自动执行 deploy.yml
    │
    ├── ① node generate-gallery.js
    │     扫描 images/ 目录 → 自动生成 photos.json
    │
    ├── ② 提交更新后的 photos.json
    │
    ├── ③ 部署 GitHub Pages（网站上线）
    │
    └── ④ 微信通知（如果配置了 PushPlus/Server酱）
         💬 "🐱 有新的猫咪照片啦！👉 点击查看"
```

**从上传到网站更新，通常只需 1-2 分钟。**

你可以打开 `https://你的用户名.github.io/仓库名` 查看最新照片。

## 7. iPhone 与 Android 区别

| 特性 | iPhone | Android |
|------|--------|---------|
| 推荐 App | PicPlus | PicHoro |
| App Store 下载 | ✅ App Store | — |
| Google Play 下载 | — | ✅ |
| 酷安下载 | — | ✅ |
| 从相册上传 | ✅ | ✅ |
| 拍照直传 | ✅ | ✅ |
| 批量上传 | ✅ | ✅ |
| 后台自动上传 | ❌ iOS 限制 | ✅ 支持 |
| 通知栏进度 | 有限 | ✅ 完整 |
| 文件命名自定义 | ✅ | ✅ |

### iPhone 特别注意

- iOS 系统限制 App 在后台运行时间，如果照片较多，**上传过程中保持 App 在前台**（屏幕不要熄）
- 如果上传一半退出 App，未完成的上传需要重新来
- 建议每次上传 5-10 张，不要一次选太多

### Android 特别注意

- PicHoro 支持后台上传，可以切到其他 App
- 部分国产手机需要在「设置」→「应用」→「启动管理」中允许 PicHoro 后台运行
- 华为/荣耀手机可能需要关闭「耗电优化」

## 8. 常见错误解决方案

### 8.1 上传报错 "401 Bad credentials"

**原因**：Token 错误或过期。

**解决**：

1. 检查 Token 是否完整复制（`ghp_` 开头，没有多余空格）
2. 如果 Token 丢失，去 https://github.com/settings/tokens 重新生成（旧的会失效）
3. 在 App 中更新 Token 后重试

---

### 8.2 上传报错 "404 Not Found"

**原因**：仓库名或用户名写错了，或者仓库不存在。

**解决**：

1. 打开 GitHub 网页，确认你的仓库名和用户名拼写正确（注意大小写）
2. 确认仓库是**公开仓库**（Private 私人仓库需要额外配置）

---

### 8.3 上传报错 "403 Resource not accessible"

**原因**：Token 权限不够。

**解决**：

1. 回到 https://github.com/settings/tokens
2. 找到你的 Token，确认 **repo** 权限已勾选
3. 如果没有，删除旧的重新生成一个，这次务必勾选 repo

---

### 8.4 上传报错 "409 Conflict"

**原因**：GitHub 上已经有同名文件。

**解决**：

1. 在 App 设置中开启「**随机文件名**」或「**时间戳命名**」
   - PicHoro：图床设置 → 高级 → 文件命名规则 → 选"日期+随机"
   - PicPlus：设置 → 上传设置 → 重命名 → 选"时间戳"
2. 这样每次上传会用不同的文件名，不会冲突

---

### 8.5 上传成功但网站没有新照片

**原因**：GitHub Actions 还没执行完，或者执行失败了。

**解决**：

1. 等 2-3 分钟再刷新网页（Actions 执行需要时间）
2. 打开 GitHub 仓库 → **Actions** 标签，查看「部署猫咪相册」是否在运行
3. 如果显示 ❌ 失败，点进去看红色步骤的错误信息
4. 常见原因：
   - `images/` 目录为空 → 确认 App 的存储路径写的是 `images/`
   - `photos.json` 格式错误 → 打开仓库检查 photos.json 内容是否正常

---

### 8.6 PicHoro 闪退

**原因**：App 版本过旧或系统不兼容。

**解决**：

1. 去酷安或 Google Play 更新到最新版
2. 尝试清除 App 缓存：手机设置 → 应用 → PicHoro → 清除缓存
3. 如果还不行，卸载重装

---

### 8.7 PicPlus 上传卡住不动

**原因**：网络问题或 Token 配置错误。

**解决**：

1. 确认手机网络正常（Wi-Fi 或移动数据）
2. 尝试切换 Wi-Fi / 移动数据
3. 删掉 App 中的 Token，重新复制粘贴一次（注意不要有多余空格）
4. 重启 App

---

### 8.8 照片太大上传失败

**原因**：GitHub 单个文件限制 100MB，但建议不超过 10MB。

**解决**：

1. 在 App 设置中开启「**压缩图片**」
   - PicHoro：上传设置 → 图片压缩 → 选"中等质量"
   - PicPlus：上传设置 → 图片质量 → 设为 85%
2. 或者在手机相册中先裁剪/压缩照片再上传
